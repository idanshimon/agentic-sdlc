Agentic SDLC v0.7

Reference design for governed agentic software development. Four planes — Standards, Pipeline, Decision Ledger, Agent HQ Runtime — sharing one audit substrate.

v0.7-rc1 62 tests passing VNET-free dev variant MIT

Why this exists

v0.6 of the agentic SDLC reference design proved one thing: governance is the differentiator, not codegen quality. Decision Ledger as audit spine, HITL gates at ambiguity classes, cost-per-decision dashboards. Two limits became visible during the workshop tour:

  1. Coverage gap. The Ledger only saw orchestrator pipeline runs — about 20% of an organization's agentic activity. Engineer IDE sessions, coding-agent issue→PR flows, reviewer Copilot Code Review runs were invisible. Compliance had audit on a fifth of the surface.
  2. Standards rigidity. Department rules (Architect, Security, Privacy, FinOps) lived as scattered prompts and APIM policies. Changing a rule meant editing code. There was no place where "Architect proposes a relaxed PHI rule" could be reviewed by Security + Privacy + Legal as a committee.

v0.7 closes both gaps with a four-plane architecture and a self-improving Pipeline Doctor that bridges them.

The four planes

1. Standards Bundles — the per-department policy plane

Architect, Security, Privacy, FinOps each own a versioned, signed bundle:

standards-bundles//v/
  ├── rules.yaml        the rules themselves
  ├── envelope.yaml     what Pipeline Doctor may auto-fix
  ├── reviewers.yaml    blast-class → reviewer roster
  └── README.md         rationale

Changes are PRs. The standards-change-agent classifies blast class, drafts an ADR, assigns reviewers per reviewers.yaml, blocks merge until quorum.

Canary rollout: 5% of teams pinned for 7 days, Pipeline Doctor watches metrics, auto-PR opens to either fully promote or revert.

2. Pipeline — the heavy lane (orchestrator)

The 9-stage governed pipeline kept verbatim from v0.6, with one substantive change:

PRD → ingest → assessor → resolver* → architect → design-review → test-plan → codegen → review-scan → deliver * HITL gate; PHI cards never autopilot.

v0.7 swap: deliver targets GitHub by default (was: ADO). ADO remains opt-in via deliver_provider="ado".

3. Decision Ledger — the audit spine (with two entry types)

Cosmos DB, team-partitioned. One container, two entry types:

runtime

Decisions made during a pipeline run, IDE session, or coding-agent flow. PHI gate APPROVED, cost $0.027, model claude-sonnet-4-6.

meta

Decisions about the rules themselves. PHI rule X relaxed for de-identified datasets, effective bundle v0.1.1, blast HIGH, 3 reviewers.

Exposed three ways: REST API, MCP server (any GitHub runtime), hook payload I/O.

4. Agent HQ Runtime Lanes — medium + light

Three lanes feed the same ledger:

LaneSurfaceWriter
heavyOrchestrator pipeline (PRD→PR)stage-by-stage
mediumGH coding agent (issue→PR from Slack/Teams/Linear/Boards)SubagentStop hook
lightVS Code Copilot, Copilot CLI5 lifecycle hooks

Hooks fire client-side (fast-path PHI guard) and write to the same Decision Ledger via the MCP server.

Hard rules — cannot be overridden

  • PHI rules NEVER auto-fixed by Pipeline Doctor. Hard-coded in envelope_validator.py — defense in depth even if a bundle's envelope.yaml is mis-edited.
  • Deny rules (severity: BLOCK or rule_pattern: "deny/*") NEVER loosened.
  • Auto-fix rate-limited to 5 / department / 7-day window.
  • Standards changes always go through committee. Pipeline Doctor proposes; humans dispose. Even on highest confidence.
  • Privacy DPO + Security Lead + Legal required reviewers for any HIGH-blast change in security/privacy bundles.

Honest disclaimers

What's working vs not

  • working Decision Ledger schema (62 tests passing across 4 components, 0 failing)
  • working PHI guard (live-tested: raw MRN → BLOCK with citation)
  • working Envelope validator (PHI hard-block + deny hard-block + bounds + preconditions + rate limits)
  • working Standards bundles (4 reference bundles validate, integration tests pass)
  • deferred ledger-insights-ui (renamed v0.6 resolver-ui not yet ported — v0.7+1)
  • deferred VNET + private endpoints (v0.7+1 — v0.6 has the proven recipe)
  • deferred True per-call cost attribution (App Insights SDK not yet wired in providers)
  • limitation GHE.com Data Residency does not yet support Coding Agent (gh.com/community/discussions/167952). EU/data-sovereignty customers limited to orchestrator pipeline lane.
  • limitation Copilot SDK is technical preview (Jan 2026). Pinned in package.json; expect API drift.
